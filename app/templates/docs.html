📦 Project Overview

This platform is an autonomous multi-agent pipeline designed to automate corporate sales development. It discovers prospects, compiles company reports using Web-scraped content and vector-based Playbook playbooks (RAG), qualifies leads, and drafts customized outreach copies.

💡 Why We Use This Tech Stack
🤝 CrewAI (Multi-Agent Cooperation)
Why: CrewAI allows setting up collaborative, role-playing agents (Research, Qualify, Copywriter) that divide and conquer tasks. It models human team structures by giving each agent specific goals, backstories, and tools.
💡 Example: The Research Agent retrieves news and writes a summary, which it then hands off directly to the Qualify Agent to calculate an Ideal Customer Profile score.
🕸️ LangGraph & LangChain (Stateful Flow Control)
Why: LangGraph manages complex, cyclical state machines and multi-step background tasks, ensuring execution order and tracking pipeline run state asynchronously. It provides built-in support for human-in-the-loop validation checkpoints.
💡 Example: Once outreach drafts are generated, the graph transitions execution state to the "human_review" node and pauses, waiting for a manager's REST or WhatsApp approval to resume.
🔎 Milvus (Vector Database / RAG)
Why: Milvus is a high-performance vector database that stores multi-dimensional embeddings of unstructured sales playbooks, case studies, and pitches. It performs rapid semantic searches to feed contextual relevance (RAG) into LLM prompts.
💡 Example: When pitching a "Robotics" company, the Copywriter agent queries Milvus to pull specific playbooks related to "Industrial Automation," avoiding generic email drafts.
🗄️ PostgreSQL (Relational Database / System of Record)
Why: PostgreSQL serves as the relational database and single source of truth for structured data, tracking user logins (RBAC), prospect lead listings, generated messages, and workflow run histories.
💡 Example: The system queries PostgreSQL to populate the dashboard leads grid with structured fields like `company_name`, `score`, and `status`.
⚙️ Platform Core Systems
LangGraph Orchestrator
Handles complex pipeline stages in the background, updating lead execution states asynchronously.
Milvus RAG Seeding
Vectorizes and stores value propositions and objection handling techniques to ground the Copywriter LLM.
🔄 Application Flow: Node Execution & Database Queries

This interactive diagram displays the LangGraph nodes. Each node represents a pipeline execution step showing exactly which databases are queried, the commands run, and the active tech stack:

1. discover_companies (Workflow Start)
FastAPI PostgreSQL
Takes target industry criteria (e.g. "Robotics") and discovers matching company prospects, storing them as raw leads.
graph LR Start([Target Industry Criteria]) --> Node[discover_companies Node] Node -->|SQL INSERT| DB[(PostgreSQL: leads table)] DB -->|Save lead status: 'discovered'| Next([Next Node])
2. research_companies (RAG Enrichment)
CrewAI Agent Milvus Vector DB PostgreSQL
Triggers the Company Intelligence Analyst Agent. Scrapes news and triggers a semantic vector search inside Milvus to retrieve relevant playbooks (RAG).
graph TD Start([Input Lead]) --> Node[research_companies Node] Node -->|1. Scrape news & growth signals| News[Web Scraping Tool] Node -->|2. Semantic search using embeddings| Milvus[(Milvus: sales_playbooks)] Milvus -->|Retrieve matching playbooks| Node Node -->|3. Compile reports| Agent[CrewAI Research Agent] Agent -->|4. SQL INSERT| DB[(PostgreSQL: research_reports table)] DB -->|Save detailed report| Next([Next Node])
3. qualify_companies (Ideal Customer Profile Scoring)
CrewAI Agent PostgreSQL
Triggers the Sales Operations Analyst Agent. Compares the research report against the manager's target criteria and calculates an Ideal Customer Profile score.
graph LR Start([Input Report]) --> Node[qualify_companies Node] Node -->|1. Ideal Customer Profile analysis| Agent[CrewAI Qualify Agent] Agent -->|2. SQL UPDATE| DB[(PostgreSQL: leads table)] DB -->|Set score & qualified status| Next([Next Node])
4. generate_outreach (Copywriting)
CrewAI Agent PostgreSQL
Triggers the Sales Copywriter Agent. Drafts a highly personalized outreach email and LinkedIn message referencing discovered growth signals and playbook value pitches.
graph TD Start([Input Qualified Lead]) --> Node[generate_outreach Node] Node -->|1. Retrieve playbooks & reports| DB_PG[(PostgreSQL & Milvus)] Node -->|2. Copywriting| Agent[CrewAI Outreach Agent] Agent -->|3. SQL INSERT| DB_Msg[(PostgreSQL: outreach_messages table)] DB_Msg -->|Save draft email & LinkedIn copy| Next([Next Node])
5. human_review (Approval Stage)
FastAPI (UI/WhatsApp) PostgreSQL
Workflow pauses. A manager reviews, edits, and approves the drafts using the Web Dashboard or Meta WhatsApp chat.
graph LR Start([Input Draft Copy]) --> Node[human_review Node] Node -->|1. Pause workflow| UI[FastAPI UI / WhatsApp webhook] UI -->|2. Edit & Approve| Node Node -->|3. SQL UPDATE| DB[(PostgreSQL: leads & messages)] DB -->|Set status: 'sent' & 'outreach_generated'| End([Workflow End])
6. Observability (LangSmith Tracing)
LangSmith
